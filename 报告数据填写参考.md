# 报告数据填写参考

本文件用于帮你把程序运行结果整理进报告。请不要把它当作最终报告直接提交，最终数据建议用本人实际运行截图核对。

## 项目差异点

方案名称：JSON 场景诊断版。

特点：从 `data/example_state.json` 读取系统状态，输出结构化 JSON 结果，便于保存测试记录和做批量案例诊断。

## 可记录数据

默认案例：

- Available = `[3, 3, 2]`
- Need 矩阵 = `[[7,4,3],[1,2,2],[6,0,0],[0,1,1]]`
- 安全性检测结果：安全
- 安全序列：`P1 -> P3 -> P0 -> P2`
- 诊断摘要：系统处于安全状态，当前检测未发现死锁风险

## 报告写法提示

- “核心实现”重点写 JSON 数据读取、Need 计算和 rounds 执行记录。
- “实验结果”可以保存 `result.json`，截图展示结构化输出。
- “AI 部分”可把 rounds 字段提供给大模型，让它按轮次解释每个进程为什么能执行。
