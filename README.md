## 方案定位

从 JSON 读取系统状态，生成执行日志，适合批量测试安全/不安全案例。

## 必做功能清单

- 读取系统状态：进程数、资源类型数、Max、Allocation、Available。
- 计算 Need 矩阵：Need = Max - Allocation。
- 完成安全性检测：输出安全/不安全；安全时给出安全序列。
- 完成请求资源模拟：检查 Request、试分配、安全检测、必要时回滚。
- 记录执行步骤：把 Need 计算、Work 更新、Finish 变化写入日志。
- 设计大模型提示词：把执行日志转成自然语言解释；不安全时给出死锁诊断建议。

## 建议完成顺序

1. 先实现并测试纯银行家算法。
2. 再实现请求资源模拟和回滚。
3. 最后接入 AI 解释模块，或者先用模拟响应替代真实 API。
4. 每完成一个功能，就把截图、运行输出和关键代码记录到报告中。

## 运行方式

请根据自己补全后的入口文件运行：

```bash
python src/json_case_advisor/run_case.py
```

## 目录说明

- `src/json_case_advisor/`：项目源代码骨架。
- `data/example_state.json`：可修改的测试状态样例。
- `PROMPTS.md`：AI 解释和死锁诊断提示词模板。
- `TEST_PLAN.md`：报告中实验结果部分的测试记录表。
